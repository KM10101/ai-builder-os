# 第一周计划

## 时间范围

- 周次：第一周
- 自然周周期：2026-03-23 至 2026-03-29
- 对应星期：周一至周日
- 计划起点：2026-03-23（周一）
- 计划终点：2026-03-29（周日）

说明：本项目的周计划统一按**自然周**计算，即每周从周一开始，到周日结束。第一周从 2026-03-23 开始，后续周计划依次顺延。

## 本周目标

这一周的目标不是学很多，而是让 AI Builder OS 真正开始运转起来。

本周结束时，希望达到以下结果：

- 明确第一阶段的学习重点
- 完成第一个最小实验
- 确定第一个小项目方向
- 建立第一次周复盘记录
- 开始形成自己的 AI coding workflow 雏形

## 本周重点主题

### 1. LLM 应用基础认知

重点理解：

- 一个最基础的 LLM 应用由哪些部分组成
- prompt 在应用中起什么作用
- structured output 为什么重要
- tool calling 在什么场景下有价值

本周不追求把所有知识讲全，而是先建立整体感。

### 2. AI 辅助开发基本工作流

重点练习：

- 如何拆解一个小需求
- 如何给 Claude Code / Codex 写清晰任务说明
- 如何审查 AI 生成的代码
- 如何通过迭代而不是一次性生成来推进开发

### 3. 第一个小应用的方向确认

这一周不一定要把项目做完，但要明确：

- 做什么
- 为什么值得做
- 这个项目能训练哪些能力
- 第一版最小可交付范围是什么

## 本周任务拆分

### 任务 1：阅读并整理基础概念

建议输出一篇简单笔记，至少回答这些问题：

- 什么是 LLM 应用
- 什么是 structured output
- 什么是 tool calling
- 什么是 workflow，什么是 agent
- 一个小型 AI 应用的最小技术结构是什么

建议产出：

- `docs/notes/llm-application-basics.md`（后续可以创建）

### 任务 2：完成第一个最小实验

实验建议：

**做一个最简单的文本总结实验**，输入一段文本，输出结构化总结结果。

目标不是做 UI，而是验证以下问题：

- 模型调用是否跑通
- prompt 是否能控制输出结构
- structured output 是否稳定
- 结果是否值得后续放进真实项目

建议产出：

- 一条实验记录
- 一段最小可运行代码

### 任务 3：确定第一个小项目

从下面几个方向里选择一个：

- 文档总结器
- 网页内容总结工具
- 笔记整理助手
- 简单知识助手

选择标准：

- 一周到两周内能做出第一版
- 能覆盖 LLM 应用基本链路
- 能练到 AI 辅助开发工作流
- 后续还能继续扩展

### 任务 4：记录 AI 工具使用过程

这一周要开始记录：

- 哪些任务适合交给 AI
- 哪些任务必须自己把关
- 什么样的提示方式更有效
- 哪些错误最容易重复出现

建议后续沉淀到：

- `docs/playbooks/ai-coding-workflow.md`

### 任务 5：完成第一次周复盘

使用周复盘模板，记录：

- 本周学到了什么
- 本周做成了什么
- 哪些地方理解还不牢
- 下一周应该继续推进什么

## 建议时间分配

如果按一周 5 到 7 天来安排，可以参考：

- 20%：看概念和整理理解
- 50%：做实验和写代码
- 20%：记录与整理
- 10%：复盘与调整

重点是避免把一整周都花在“看资料”上。

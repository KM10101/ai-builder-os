## 实践 / 学习记录

### 今日关注重点

今天不直接默认进入功能迭代，而是先解决两个问题：

1. 后续实验是否已经有一个稳定可用的开发环境？
2. AI coding workflow 是否应该显式包含“实验前检查”这一步？

如果环境和工具链不稳定，那么 workflow 本身的实验结果会被噪音干扰，难以准确判断问题到底来自 prompt、流程还是环境。

### 建议优先关注的问题

- WSL 是否已经可以替代原来的远程 Linux 开发方式，或者至少成为默认开发环境
- Codex / AI coding 工具连接不稳定的问题，是否有可重复出现的触发条件
- 权限确认策略是否需要在实验前统一配置
- 抓取目标网页时，如何提前判断页面是否依赖 JS 渲染，避免把网站特性误判为实现问题

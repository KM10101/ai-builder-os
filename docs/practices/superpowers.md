# Superpowers 实践笔记

## 这份文档用来做什么

用于记录你对 superpowers 的理解、尝试和吸收。

当前阶段先不急着做自己的 superpowers，而是重点关注：
- 它解决了什么问题
- 它抽象了哪些高频任务
- 哪些设计适合借鉴
- 哪些模式适合纳入自己的系统

## 建议记录角度

### 1. 它在抽象什么

比如：
- 常见任务入口
- prompt 复用
- 流程封装
- 上下文组织

### 2. 哪些地方值得学

- 是否降低了起手成本
- 是否减少了每次从零写 prompt
- 是否把复杂流程变成可复用模式

### 3. 哪些地方暂时不要照搬

- 过早工具化
- 绑定某个固定工作流
- 把还不稳定的方法过早封装

## 你的记录模板

- 我看到了什么做法
- 这背后解决的是什么问题
- 哪些点适合我当前阶段
- 哪些点适合以后再考虑
- 我可以先沉淀成 pattern / prompt / case 的是什么

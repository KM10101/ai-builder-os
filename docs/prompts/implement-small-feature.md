# 小功能实现模板

## 适用场景

当你要实现一个明确、边界较清晰的小功能时使用。

## 模板

```text
我想实现 XXX。

先不要直接写代码，请先输出：
1. 需要修改哪些文件
2. 每个文件大概要改什么
3. 有没有兼容性风险
4. 测试应该怎么补

等我确认后，再开始修改。
```

## 执行阶段补充模板

```text
按刚才确认的方案修改代码，并满足以下要求：
- 尽量少改动
- 保持现有代码风格
- 不要顺手做无关重构
- 改完后总结修改点
- 列出需要我重点 review 的地方
- 如果能跑测试，告诉我建议执行哪些验证命令
```

## 使用建议

- `XXX` 尽量写成具体结果，不要太抽象
- 如果任务有边界，明确说只改哪些目录 / 模块
- 如果不希望它扩散，记得加“不要顺手重构”

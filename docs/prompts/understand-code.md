# 代码理解模板

## 适用场景

当你需要先理解代码，不希望工具立即修改时使用。

## 模板

```text
先不要修改代码。

请帮我阅读这个项目里和 XXX 相关的代码，完成下面几件事：
1. 找出入口文件和关键调用链
2. 用简洁的话解释当前实现逻辑
3. 标出最关键的 3 个文件
4. 说出你认为最容易出问题的点

如果信息不够，先告诉我还需要看哪些文件。
```

## 使用建议

- 把 `XXX` 替换成具体模块、接口、功能点
- 可以补充“只看某几个目录”来缩小范围
- 如果仓库很大，先限定文件夹或服务边界

## 常见追加约束

- 先不要给解决方案
- 只关注当前问题相关代码
- 输出尽量简洁
- 如果有不确定的地方请明确说出来

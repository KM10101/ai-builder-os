## 示例填法（参考）

### 背景
我已经完成了一版接口改动，但不确定边界条件和兼容性有没有问题，希望先让 AI 做一轮预审。

### 使用的工具
主要使用 Claude Code，因为这次更偏向风险识别和 review。

### 使用的 Prompt
请 review 这次改动，重点看：
- 是否有逻辑漏洞
- 是否会影响已有功能
- 是否缺少边界条件处理
- 是否需要补测试
- 是否存在可以简化的实现

请优先给高风险问题，不要泛泛而谈。

### 有效点
- 要求“优先高风险问题”后，输出明显更有价值
- 对 review 场景来说，Claude Code 的作用比直接执行修改更大

### 下次优化
下次可以附上更明确的 diff 范围和改动背景，减少泛泛建议。

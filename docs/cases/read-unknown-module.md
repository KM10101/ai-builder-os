## 示例填法（参考）

### 背景
接到一个需求，要调整订单超时取消逻辑，但我对当前服务不熟，先需要搞清楚代码入口和调用链。

### 工具选择
主要使用 Claude Code，因为这次目标是先理解现状，而不是立即修改代码。

### 使用的 Prompt
先不要修改代码。

请帮我阅读这个项目里和订单超时取消相关的代码，完成下面几件事：
1. 找出入口文件和关键调用链
2. 用简洁的话解释当前实现逻辑
3. 标出最关键的 3 个文件
4. 说出你认为最容易出问题的点

### 有效点
- Claude Code 在梳理调用链和总结关键文件上很有帮助
- “先不要改代码”这句约束非常重要
- 要求它给出风险点，比只要代码解释更有价值

### 下次优化
下次可以一开始就补充服务目录范围，避免它扫描过多无关模块。

## 示例填法（参考）

### 背景
需要给某个接口增加一个 `source` 字段透传，需求不复杂，但不希望 AI 工具改动范围扩散。

### 工具分工
- 先让 Claude Code 输出修改计划
- 再让 Codex 按方案执行

### 使用的 Prompt
我想实现给订单接口增加 `source` 字段透传。

先不要直接写代码，请先输出：
1. 需要修改哪些文件
2. 每个文件大概要改什么
3. 有没有兼容性风险
4. 测试应该怎么补

### 有效点
- 先看计划再执行，明显降低了改偏概率
- “不要顺手重构”很重要
- 改完后要求总结修改点，便于复查

### 下次优化
下次可以更明确指定“只允许改 controller / service / DTO”之类的文件范围。

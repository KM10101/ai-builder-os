# Claude Code 常用命令速查

> 这页优先服务于“快速上手 + 实战速查”，不追求覆盖全部能力，而是先覆盖高频有用内容。

## Claude Code 更适合做什么

默认更适合优先承担：

- 读代码
- 梳理调用链
- 拆任务
- 给改动方案
- 做代码 review
- 帮你识别风险点

## 高频使用方式

### 1. 在项目目录启动

```bash
claude
```

建议进入项目后第一句话不要直接让它改代码，而是先做理解。

### 2. 读代码模式

典型起手：

```text
先不要修改代码，先帮我理解这个仓库里和 XXX 相关的实现。
```

适合：
- 接手陌生模块
- 找入口
- 看调用链
- 理解现状

### 3. 方案模式

典型起手：

```text
我想实现 XXX。先不要直接写代码，请先告诉我需要修改哪些文件、每个文件要改什么、风险点是什么。
```

适合：
- 小需求改动前先收敛范围
- 改动已有逻辑前先看影响

### 4. Review 模式

典型起手：

```text
请 review 这次改动，优先指出高风险问题、边界条件遗漏和潜在兼容性问题。
```

适合：
- 自己改完后复查
- 看别人的改动
- PR review 前预审

## 推荐高频 prompt 片段

### 先理解，不动代码

```text
先不要改代码。
```

### 限定范围

```text
只看和当前问题直接相关的模块，不要扩展到无关部分。
```

### 输出关键文件

```text
请标出最关键的 3 个文件，并解释各自作用。
```

### 要求风险分析

```text
请指出你认为最容易出问题的点。
```

### 要求先出计划

```text
先输出修改计划，等我确认后再执行。
```

## 实战建议

### 适合 Claude Code 的默认节奏

1. 先理解现状
2. 再输出方案
3. 最后 review 执行结果

### 常见坑

- 一上来就让它直接实现
- 没有限定分析范围
- 没要求它指出风险点
- 对大任务没有先拆小

## 一个推荐定位

把 Claude Code 当成：

- **代码理解助手**
- **任务拆解助手**
- **高级 review 搭子**

而不是默认把它当“全自动写代码工具”。

## 示例用法

### 示例 1：阅读陌生接口

```text
先不要修改代码。帮我阅读和订单超时取消相关的实现：
1. 找入口文件和关键调用链
2. 解释当前处理逻辑
3. 标出最关键的 3 个文件
4. 说出最容易出问题的点
```

### 示例 2：改动前先做方案

```text
我想给这个接口增加一个状态字段。先不要直接写代码，请先输出：
1. 要修改哪些文件
2. 每个文件大概要改什么
3. 有没有兼容性风险
4. 测试应该怎么补
```

### 示例 3：做一次 Review

```text
请 review 这次改动，重点看：
- 是否有逻辑漏洞
- 是否遗漏边界条件
- 是否有兼容性风险
- 是否需要补测试
请优先给高风险问题。
```

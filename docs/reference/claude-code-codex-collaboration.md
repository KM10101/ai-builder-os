# Claude Code / Codex 协作速查

## 一个默认可用的协作链路

当你不确定怎么分工时，可以先用这个默认套路：

1. **Claude Code**：理解代码、梳理现状、给修改方案
2. **Codex**：按确认后的方案执行修改
3. **Claude Code**：对改动做 review 和查漏补缺

## 适合这样配合的场景

- 你对代码还不够熟
- 任务不算大，但有一定改动风险
- 你希望“先收敛，再落地，再复查”

## 快速起手模板

### 第一步：交给 Claude Code

```text
先不要改代码，帮我理解 XXX，并给出修改方案、影响范围和风险点。
```

### 第二步：交给 Codex

```text
按确认后的方案修改代码，尽量少改动，不要顺手重构，改完后总结修改点和风险点。
```

### 第三步：再交给 Claude Code

```text
请 review 这次改动，优先指出高风险问题、边界条件遗漏和是否需要补测试。
```

## 什么时候不必双工具

下面这些情况直接单工具就够：

- 只是简单读代码 → 优先 Claude Code
- 只是明确小改动 → 优先 Codex
- 只是快速 review → 优先 Claude Code

## 核心原则

- Claude Code 偏理解与判断
- Codex 偏执行与落地
- 真正重要的是任务边界清晰，而不是机械分工

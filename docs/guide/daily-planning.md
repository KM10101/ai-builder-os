# 每日计划与执行记录

## 目的

为了让 AI Builder OS 的学习—实践—复盘循环真正稳定运转，从第一周开始引入按天记录的机制。

每日文档用于同时承担两件事：

1. 记录当天计划完成的内容
2. 记录当天实际完成情况、结果和反思

这样做的目的不是增加形式，而是帮助自己：

- 把每一天的学习主题和实践动作连接起来
- 避免只有周计划、没有日落地
- 避免做完之后没有记录，导致经验无法沉淀
- 为周复盘提供清晰输入

## 使用方式

每一天使用一个独立文档。

命名格式：

- `YYYY-MM-DD.md`

建议按周归档，例如：

- `docs/daily/2026-week-13/2026-03-24.md`
- `docs/daily/2026-week-13/2026-03-25.md`

说明：

- 周编号按自然周理解
- 一周内的每日计划和记录放在同一个周目录下
- 周复盘仍然单独放在 `docs/reviews/`

## 每日文档建议结构

```md
# 2026-03-24

## 今日主题

## 今日目标

## 今日计划
- [ ]
- [ ]
- [ ]

## 实践 / 学习记录

## 今日产出
- 文档：
- 代码：
- 实验：

## 遇到的问题

## 今日结论 / 反思

## 完成情况
- 完成度：
- 是否需要延续到明天：
```

## 编写原则

### 1. 计划不要写太多

每日计划尽量控制在 2 到 4 项。

重点不是把任务列满，而是保证当天真正能推进，并且能在结束时完成记录。

### 2. 计划和记录写在同一个文件里

这样更容易看出：

- 原计划是什么
- 实际完成了什么
- 偏差出现在哪里

### 3. 每天至少留下一个“有效结论”

哪怕当天推进不大，也尽量留下至少一条：

- 对概念的理解
- 对实验结果的判断
- 对 AI coding workflow 的观察
- 对下一步任务边界的收缩

### 4. 为周复盘服务

每日文档不是独立存在的，它应该成为周复盘的原始材料。

到周末时，可以直接根据这几天的每日文档整理：

- 学到了什么
- 做成了什么
- 卡在了哪里
- 下周该怎么调整
```

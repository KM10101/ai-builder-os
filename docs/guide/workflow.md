# 使用方式

这个仓库的目标不是堆文档，而是帮助你在真实工作中快速复用经验。

## 推荐工作流

### 1. 做任务前：先查
先到这些目录找现成内容：

- `patterns/`：看这类任务通常怎么做
- `prompts/`：复制一个合适的 prompt 起手
- `reference/`：快速查命令或常用片段

### 2. 做任务时：边用边试
在真实任务中使用 Claude Code / Codex：

- 不追求一把梭
- 优先用“先理解 → 再方案 → 再执行”
- 把有效提示词和失败经验先随手记下

### 3. 做完任务后：补案例
把这次任务整理到 `cases/`：

- 背景
- 使用的工具
- 用过的 prompt
- 哪些地方有效 / 无效
- 下一次可以如何做得更好

### 4. 出现重复后：提炼
当你发现某种做法重复出现时：

- 把有效 prompt 提炼到 `prompts/`
- 把稳定方法抽象到 `patterns/`
- 当流程已经成熟时，再考虑放进 `skills/`

## 一个简单判断

### 什么时候写到 `prompts/`
当你发现：
- 某句 prompt 很常用
- 只要稍微改一下就能复用
- 主要价值在于“快速起手”

### 什么时候写到 `patterns/`
当你发现：
- 不只是 prompt 本身有用
- 更重要的是背后的做法和判断顺序
- 已经能总结出适用场景与边界

### 什么时候写到 `skills/`
当你发现：
- 这已经不是一句 prompt，而是一套流程
- 有明确输入、步骤、输出
- 值得长期复用和进一步封装

## 当前最重要的原则

**先沉淀真实使用经验，再抽象方法；先积累内容，再考虑造工具。**

# 项目结构

AI Builder OS 现在采用“内容分层”的方式组织，以支持长期积累与快速复用。

## 目录结构

```text
docs/
├── guide/          # 如何使用这个系统
├── patterns/       # 方法论 / 执行模式
├── prompts/        # 可复制提示词模板
├── practices/      # 对外部优秀实践的学习记录
├── reference/      # 常用命令、速查内容
├── cases/          # 真实任务案例与复盘
├── skills/         # 已成熟 / 待封装的复杂能力
├── playbooks/      # 跨模式清单、执行规范
├── templates/      # 各类记录模板
├── system/         # 项目的设计原则与定位
└── .vitepress/     # 文档站配置
```

## 每层分别解决什么问题

### `patterns/`
解决：**这类任务通常应该怎么做？**

适合放：
- 方法论
- 工具分工策略
- 任务处理套路
- 使用边界与注意事项

### `prompts/`
解决：**这类任务我应该怎么起手提问？**

适合放：
- Claude Code 提示词模板
- Codex 提示词模板
- 共享 prompt 片段
- 限定范围 / 先出方案 / 自查等约束模板

### `practices/`
解决：**外部优秀实践里有什么值得吸收？**

适合放：
- superpowers
- get-shit-done
- oh-my-codex
- 其他值得借鉴的 workflow

### `reference/`
解决：**我现在就想快速查一下命令 / 语法 / 快捷套路。**

适合放：
- Claude Code 常用命令
- Codex 常用命令
- 常用 prompt 片段速查
- AI coding 常见检查项

### `cases/`
解决：**这次真实任务我是怎么做的？哪里做得好，哪里踩坑了？**

适合放：
- 工作案例
- 学习实验案例
- prompt 成败记录
- 工具协作复盘

### `skills/`
解决：**哪些复杂流程已经成熟，值得封装成 skill？**

适合放：
- skill 想法
- 升级标准
- 待封装流程
- 已稳定能力说明

## 推荐演进路径

建议遵循：

1. 先记 `cases`
2. 再提 `prompts`
3. 再抽 `patterns`
4. 最后进 `skills`

## 实用原则

- 内容先求可用，不求完美
- 先记录最小有用单位
- 一篇文档只解决一个核心问题
- 模板和案例一定要能被未来的自己快速理解

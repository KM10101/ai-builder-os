# 运转循环

## 主循环

AI Builder OS 按照一个重复循环来运转：

1. 选择当前重点方向。
2. 学习核心理论。
3. 做一个最小实验。
4. 在项目中应用一次。
5. 做结果复盘。
6. 更新路线图。

## 输入

系统常见输入包括：

- 一个新的学习主题
- 一个值得验证的项目想法
- 实现过程中暴露的问题
- 复盘时发现的能力缺口
- 一个值得评估的新工具或框架

## 输出

系统常见输出包括：

- 概念笔记
- 实验记录
- 可复用代码
- 工作流模式
- 项目产物
- 周复盘和月复盘

## 复盘问题

复盘时可以重点回答这些问题：

- 我学到了什么？
- 我做成了什么？
- 哪些地方比之前更清楚了？
- 哪些地方仍然薄弱？
- 哪些内容以后可以复用？
- 下一步最值得投入的重点是什么？

## 维护原则

如果系统中的某一部分已经不再帮助成长，就应该简化它，或者删除它。

这个系统存在的目的是支持成长，而不是制造额外负担。

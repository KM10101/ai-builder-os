# 初始能力基线

## 这份文档的作用

这不是为了评价自己“强不强”，而是为了在项目开始时，对当前能力状态有一个更清晰的基线判断。

后面每隔几周都可以回来看一次，确认哪些能力真的在增长。

## 评估方式

建议每个能力项按下面 4 档简单自评：

- 1：了解很少，基本不能独立完成
- 2：知道基本概念，但做起来不稳定
- 3：可以独立完成基础实现
- 4：可以稳定实现并解释关键取舍

## 使用建议

这份基线不要追求绝对准确，只要足够真实就行。

最重要的是：

- 不要高估“看过”等于“会做”
- 不要低估已经有工程经验的部分
- 每次复盘时关注变化，而不是分数本身

后续可以在阶段结束时新增一份对比版文档，用来观察能力变化轨迹。

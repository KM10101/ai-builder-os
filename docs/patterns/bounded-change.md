# 小步修改，限制范围

## 适用场景

适用于：
- 修一个明确 bug
- 加一个小字段 / 小逻辑
- 补一个测试
- 对已有模块做低风险迭代

## 核心思想

AI 编程工具在边界模糊时容易扩散改动。

因此应主动限制：
- 改动目标
- 改动文件范围
- 不要顺手重构
- 完成后自查风险

## 推荐约束

- 尽量少改动
- 只改和本需求直接相关的文件
- 不要顺手做无关重构
- 保持现有代码风格
- 改完总结修改点和风险

## 推荐步骤

1. 先明确本次目标
2. 先让工具列出计划改哪些文件
3. 确认后再执行
4. 执行后要求自查和验证建议

## 常见坑

- 目标描述太泛
- 没有指定不要做额外重构
- 改完后没有复查风险
